## Upstream Collaboration — buffyboard Numeric Keyboard

Significant progress on the buffyboard feature request filed in Session 3. The issue attracted responses from two postmarketOS contributors:

**Johannes Marbach (Maintainer)** responded to the auto-detection suggestion:
- Confirmed a numeric layout in `sq2lv_layouts.c` is feasible
- Noted the challenge: password type needs to be persisted somewhere buffyboard can read before LUKS is unlocked
- Suggested saving a flag and passing it via CLI at runtime

**Vladimir Stoiakin (Contributor)** proposed an elegant solution:
- LUKS2 token area already supports storing arbitrary JSON metadata
- systemd already uses this mechanism to distinguish recovery keys from regular passphrases
- Flow: pmbootstrap detects password type → writes JSON token to LUKS2 slot → buffyboard reads token at boot → selects appropriate keyboard layout
- No separate file needed, already part of the LUKS2 spec

**Connor's reply** summarised the proposed flow and asked whether buffyboard currently has LUKS2 token reading capability or whether that needs to be added alongside the numeric layout.

**Status:** Active technical discussion, implementation path becoming clear. This feature will benefit all postmarketOS touch device users with numeric LUKS passphrases.

## Setup Wizard — First Working Build 🎉

The first working build of the MahalaOS setup wizard was completed and deployed to the OnePlus 6T tonight. This is MahalaOS's biggest consumer-facing milestone to date — no other mobile Linux distribution has a first-boot setup experience.

### Tech Stack
- Python + GTK4 + libadwaita
- Navigation via GtkStack — consistent with other mobile setup wizards
- Deployed and tested on device over USB SSH

### Screens Implemented

Seven screens covering the full first-boot flow:

1. **Welcome** — MahalaOS branding, friendly intro
2. **Language & Region** — locale selection with clean reboot flow
3. **WiFi** — fully implemented network scanning and connection
4. **SIM Detection** — carrier detection via mmcli
5. **What to Expect** — honest plain-English feature status breakdown
6. **WhatsApp/Waydroid Setup** — guided Android app setup (stub)
7. **Completion** — you're ready screen

### Key Technical Achievements

**WiFi screen** is fully implemented via nmcli:
- Detects already-connected networks
- Lists available networks by signal strength
- Handles password entry
- Only unlocks Continue once connection is verified

**SIM detection** uses mmcli with a hard 8-second timeout — never hangs regardless of modem state.

**Language reboot flow** — if user selects a different language, Continue switches to "Restart now" in their chosen language and reboots the device.

**Resume mechanism** — a resume file at `/var/lib/mahalaos/wizard-resume-step` ensures:
- Post-language-reboot: wizard resumes at WiFi screen, not Welcome
- Crash or unexpected shutdown: wizard resumes at last completed step
- Users never lose progress

**Honest screen** — plain-English breakdown of what works, what partially works, and what doesn't yet. No marketing spin. Builds trust at first boot.

### Remaining Stubs
- Waydroid launch on WhatsApp screen
- Expanding language list

### Significance

This is the "Argos test" feature — the thing that makes MahalaOS different from every other mobile Linux project. A normal person turning on this phone for the first time now gets guided through WiFi, SIM setup, and an honest explanation of what the OS can and can't do. Nobody else in the mobile Linux space has this.

It will be the centrepiece of the NLnet application.

## Strategic Planning Notes

Several strategic directions discussed and documented for future reference:

**Distribution model:**
- Repair shop franchise/installer program — shops charge customer for install, MahalaOS receives small per-device fee
- CEX identified as strong potential partner — trusted brand, technical staff, constant supply of second hand devices, UK coverage
- Framing: "Pay for privacy, installed for you" rather than "pay for an OS"

**Revenue model:**
- App store/OEM licensing: 75% MahalaOS, 12.5% postmarketOS, 12.5% discretionary (Linux Foundation, GNOME, FSF)
- OEM licensing as first real revenue source
- NLnet grant as bridge funding

**NLnet strategy:**
- Allocate percentage of funding explicitly to upstream postmarketOS development
- Pick 2-3 milestones, fund them properly, deliver completely
- Track record of completed milestones strengthens subsequent applications

**Future features (parked):**
- MahalaOS-to-MahalaOS native features (secure messaging, contact indicators, local network sharing)
- Cross-distro compatibility for native features
- First-boot setup wizard (briefing document prepared for dedicated session)
